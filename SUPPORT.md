# Support

Thanks for using Covalida. Here is how to get help, routed to the fastest channel
for each kind of request.

> All contact addresses and URLs below are maintained canonically in
> [`docs/RESOURCES.md`](docs/RESOURCES.md).

## Where to go

| I want to… | Go to |
| --- | --- |
| Read product & API documentation | [docs.covalida.com](https://docs.covalida.com) |
| Ask a question or share an idea | [GitHub Discussions](https://github.com/orgs/covalida/discussions) |
| Report a bug or request a feature | Open an issue in the relevant repository |
| Reach us for general or business matters | [hello@covalida.com](mailto:hello@covalida.com) |
| Ask about compliance, accreditation or EC-REP | [compliance@covalida.com](mailto:compliance@covalida.com) · [covalida.com/s/ec-rep](https://covalida.com/s/ec-rep) |
| Report a security vulnerability | **Do not open a public issue** — see [SECURITY.md](SECURITY.md) → [security@covalida.com](mailto:security@covalida.com) |
| Review certifications & sub-processors | [Trust Center](https://trust.covalida.com) |

## Before opening an issue

1. Check the [documentation](https://docs.covalida.com).
2. Search existing issues and [Discussions](https://github.com/orgs/covalida/discussions)
   to avoid duplicates.
3. Use the issue template that best fits your request.

## What is not handled via issues

- **Security vulnerabilities** — follow the [Security Policy](SECURITY.md).
- **Code of Conduct concerns** — see [conduct@covalida.com](mailto:conduct@covalida.com)
  in the [Code of Conduct](CODE_OF_CONDUCT.md).
- **Commercial / certification enquiries** — email
  [compliance@covalida.com](mailto:compliance@covalida.com) or use the
  contact form at [covalida.com](https://covalida.com).
